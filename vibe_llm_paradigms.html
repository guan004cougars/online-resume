LLM 范式平台

按「LLM 在产品里扮演什么角色」切分出 toC 7 种 + toB 7 种 范式。每张卡片背后是一个真实调用大模型的可玩 demo + 一页产品方法论。

👤 面向消费者 · toC
模型在产品里扮演什么角色 —— 从「模型即界面」到「模型是隐形引擎」
💬
C1
对话即产品
Chat-native ・ 模型即界面
LLM 本身就是产品。用户自带任务，直接对话。代表：ChatGPT、Claude、豆包。

🫂
C2
角色陪伴
Companion ・ 卖情感关系
把模型包装成有人格的角色，卖陪伴而非效率。代表：Character.AI、星野。
进入 demo →方法论
🎨
C3
创作工具
Co-creation ・ 要产出物
模型作为生产力工具，用户要的是可反复打磨的产出。代表：Notion AI、文案/图像生成。
进入 demo →方法论
✍️
C4
嵌入式 Copilot
Copilot ・ 增强已有操作
把 AI 塞进已有高频场景，0 迁移成本。代表：浏览器侧栏、文档助手、输入法 AI。
进入 demo →方法论
🤖
C5
任务代理 Agent
Agent ・ 替你把事办完
给目标，模型自己拆解、分步执行、交付结果。代表：Manus、Deep Research。
进入 demo →方法论
🔍
C6
答案引擎
AI-native rebuild ・ 用 AI 重做老品类
AI 藏在背后，对标某个老品类拉开体验代差。代表：Perplexity 重做搜索。
进入 demo →方法论
🔀
C7
聚合 / 路由层
Meta layer ・ 卡在用户与模型间
聚合多模型做分发与对比，收过路费。代表：Poe、各类全家桶入口。
进入 demo →方法论
🏢 面向企业 · toB
把大模型能力嵌入企业工作流 —— 知识、副驾、客服、流程、分析、文档、平台
🗄️
B1
企业知识库问答
Enterprise RAG ・ 让企业文档可问答
把散落的企业文档变成可问答、可溯源的知识中枢。代表：Glean、各类知识中台。

Vertical Copilot ・ 嵌入业务系统

智能客服
AI Support ・ 自动应答+坐席辅助
知识驱动的自动应答 + 坐席实时辅助，可溯源、可转人工。代表：各家 AI 客服。
