# 分析目标指引

不同的竞品分析目标，会决定不同的分析重点。开始分析前，先判断本次任务属于哪一类。

## 1. 新产品立项

适用场景：
- 用户想判断一个方向值不值得做
- 用户想了解赛道是否拥挤、是否还有机会
- 用户想知道现有玩家是如何分层竞争的

重点关注：
- 市场关键竞争要素
- 市场格局
- 主要玩家分层
- 用户核心痛点是否已被充分满足
- 当前产品的共同模式
- 尚未被满足的机会点
- 新进入者的切入难度

输出重点：
- 赛道是否值得进入
- 该市场最核心的竞争因子是什么
- 更适合从哪个人群或场景切入
- 哪些方向已经过于拥挤
- 哪些差异化路径值得优先探索

## 2. 功能规划

适用场景：
- 用户准备做某个功能
- 用户想比较不同产品是怎么设计同类功能的
- 用户想判断哪些功能值得跟进

重点关注：
- 该功能所在环节的关键竞争要素
- 功能覆盖情况
- 关键任务流程
- 交互方式
- 功能深度与易用性
- 用户完成目标的路径是否顺畅
- 不同产品对同一问题的不同解法

输出重点：
- 哪些能力属于基础补齐项
- 哪些能力属于差异化项
- 哪些设计值得参考
- 哪些做法不建议直接照搬
- 哪些判断已有数据或证据支撑

## 3. 定价或商业模式评估

适用场景：
- 用户想研究收费方式
- 用户想判断免费、订阅、套餐、增值服务的设计逻辑
- 用户想对比不同产品如何完成商业转化

重点关注：
- 市场对价格的敏感度
- 价格体系
- 套餐结构
- 免费与付费边界
- 核心付费点
- 转化路径
- 商业模式与目标用户是否匹配

输出重点：
- 各竞品靠什么收费
- 收费设计是否与产品价值一致
- 哪些定价策略更适合当前用户群
- 我方可借鉴或需规避的模式
- 哪些价格判断已有公开数据或可信证据支撑

## 4. 增长策略参考

适用场景：
- 用户想了解竞品如何获客
- 用户想研究内容、渠道、裂变、品牌打法
- 用户想分析竞品增长路径

重点关注：
- 增长竞争的关键要素是什么
- 获客渠道
- 注册转化路径
- 首次价值体验
- 内容策略
- 裂变与分享机制
- 品牌表达与传播切口

输出重点：
- 各竞品主要增长抓手是什么
- 哪些增长方式与产品形态强相关
- 哪些方式值得测试
- 哪些方式不适合我方阶段
- 哪些增长判断需要更多数据验证

## 5. 管理层或干系人汇报

适用场景：
- 用户需要快速向上汇报
- 用户要整理一份结论先行的竞品材料
- 用户要帮助团队统一认知

重点关注：
- 结论是否清楚
- 数据是否可信
- 市场格局是否易懂
- 差异点是否明确
- 建议是否可执行
- 内容是否便于快速浏览

输出重点：
- 市场结论
- 关键竞争要素判断
- 核心竞品差异
- 重要企业路径变化
- 对我方的主要启发
- 建议的下一步动作

## 使用原则

如果用户没有明确说明目标：
- 优先从用户当前任务推断目标
- 若用户提到“立项”，优先按新产品立项处理
- 若用户提到“功能”，优先按功能规划处理
- 若用户提到“收费”，优先按定价评估处理
- 若用户提到“增长”，优先按增长策略处理
- 若用户提到“汇报”，优先按管理汇报处理

如果一个任务同时包含多个目标：
- 先确定主目标
- 次目标只作为补充
- 不要让输出失去重点
