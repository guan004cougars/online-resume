# 市场竞争格局分析

这个文件用于指导：如何把“市场竞争格局”做成一个真正有判断价值的模块，而不是只列几家竞品名称。

## 一、这部分要回答什么

市场竞争格局的目标，不只是告诉用户“市场上有哪些玩家”。

更重要的是回答：
- 这个市场到底是按什么方式竞争的
- 不同企业、产品或业务形态分别占据什么位置
- 谁是头部，谁是中部，谁是长尾
- 哪些是直接竞品，哪些是替代方案
- 市场还有没有空位，空位在哪

## 二、默认应包含哪些内容

如果用户要做系统型竞品分析，建议把“市场竞争格局”单独成节。

默认可包含这几类内容：
1. 二维四象限散点图
2. 市场份额饼图或分布图
3. 头部、中部、长尾分层
4. 直接竞品、替代方案、业务形态映射
5. 市场空位与竞争空白
6. 进入门槛与竞争趋势

不是每次都要全部展开，但至少建议包含：
- 一个清晰的“坐标轴 + 点”的二维散点图
- 一个市场份额饼图或分层判断

## 三、二维四象限怎么选维度

二维四象限的关键，不是为了画图，而是为了把市场的主要竞争方式看清楚。

更推荐的表现形式不是四个大框，而是：
- 中间有 X / Y 轴
- 每个企业、产品或业务形态用一个点表示
- 点位直接反映其在两个维度上的相对位置

这样比简单按象限列名字，更接近真实判断。

### 一个强约束

x 轴和 y 轴，都应从“市场关键竞争要素”中选择，而不是另起一套维度体系。

也就是说，正确顺序应该是：
1. 先识别赛道的市场关键竞争要素
2. 再从这些竞争要素里选 2 个最有区分度的，作为 x / y 轴
3. 再把企业、产品或业务形态放到图上

不要反过来先想画图，再临时发明两个轴。

维度选择原则：
- 选真正影响用户选择的核心竞争要素
- 两个维度尽量彼此独立
- 能够把玩家明显拉开
- 能够反映该赛道最重要的取舍关系
- 轴的来源必须可追溯到前面已经识别出的市场竞争要素

### 轴的选择会随分析对象变化

同一个行业里，即使竞争要素集合相同，最终选哪两个要素做坐标轴，也可能因为分析对象不同而变化。

例如电商行业常见竞争要素是：
- 多
- 快
- 好
- 省

如果分析综合电商、即时零售、本地商超和便利店，常更适合选：
- 多 x 快

如果分析品牌零售、高客单零售和精选电商，可能更适合选：
- 好 x 省
或
- 好 x 多

所以要记住两点：
- 轴的来源必须来自市场竞争要素
- 轴的最终选择，要根据本次研究的产品、企业和分析目标来决定

不建议选：
- 过于抽象的维度
- 两个高度重叠的维度
- 只适合单个产品、无法覆盖整个市场的维度
- 明明不在市场竞争要素里，却为了画图好看硬凑出来的维度

## 四、常见维度选择方式

可优先从这些角度选：
- 供给丰富度 vs 履约速度
- 标准化程度 vs 定制化程度
- 通用能力 vs 垂直深度
- 轻交付 vs 重交付
- 价格水平 vs 服务深度
- 单点效率 vs 全流程闭环
- 自助式使用 vs 专业服务依赖
- C 端覆盖度 vs B 端能力深度

## 五、示例：电商赛道

如果从“物品丰富度”和“送达时间”两个维度看电商市场：

- 高丰富度 + 快送达：
  京东次日达、即时零售平台中的核心仓配模式

- 高丰富度 + 慢送达：
  综合电商平台、跨区域供给平台

- 低丰富度 + 快送达：
  家门口小超市、1 公里内便利店、社区即时零售

- 低丰富度 + 中等或偏慢送达：
  垂直小店、非标准化本地供给

这类图的意义在于：
- 用户能看到不同业务形态其实也在争同一需求
- 不只是比较“平台 A 和平台 B”
- 还能看出哪些位置已经拥挤，哪些位置仍有空位

## 六、市场份额分布图怎么做

如果有公开数据，建议补“市场份额饼图”或“市场份额分布图”。

可以用这些方式：
- 饼图思路：看整体份额占比
- 条形图思路：看各玩家相对大小
- 分层图思路：看头部、中部、长尾结构

重点不是图形本身，而是给出判断：
- 市场是否高度集中
- 是否存在寡头
- 中腰部是否还有成长空间
- 长尾是否大量分散

所以在“市场份额分布”下面，建议默认再补一句定性判断，例如：
- 当前更接近寡头型市场
- 当前属于分散型市场
- 当前属于长尾分散型市场
- 当前尚未形成稳定寡头结构

最好进一步说明为什么这样判断，而不只是给一个标签。

必须标清：
- 数据时间
- 数据口径
- 数据来源
- 是否为估算

如果没有统一口径，不要硬做精确份额图，可以改为：
- 头部 / 中部 / 长尾分层判断
- 份额趋势判断
- 下载量、用户量、覆盖医院数、营收规模等替代指标

## 七、还可以放哪些内容

为了让用户更完整地了解市场，这一部分还可以补：

### 1. 玩家分层
- 头部玩家
- 中腰部玩家
- 长尾和区域玩家

### 2. 业务形态地图
- 平台型
- 工具型
- 解决方案型
- 渠道型
- 线下替代形态

### 3. 直接竞品与替代方案
- 谁是同赛道直接竞争
- 谁其实在替代同一核心需求

### 4. 市场空位
- 哪个象限拥挤
- 哪个象限空白
- 哪些用户需求尚未被充分满足

### 5. 进入门槛
- 渠道门槛
- 牌照或合规门槛
- 供应链门槛
- 交付和服务门槛
- 数据和技术门槛

### 6. 竞争趋势
- 竞争正从什么转向什么
- 头部企业在加什么能力
- 中小企业靠什么存活

## 八、适合写在结论里的判断

更好的表达方式：
- 当前市场不是简单按品牌竞争，而是按“供给丰富度 x 履约时效”形成不同业务形态分布
- 头部玩家集中在高丰富度、高效率象限，中腰部更多依靠垂直深度或区域能力生存
- 这个市场的替代方案很多，直接竞品压力不一定最大，跨形态替代反而更强
- 市场份额已呈现头部集中，但细分场景仍存在局部空位
- 未来竞争可能从单点能力转向全流程闭环

不够好的表达方式：
- 市场竞争激烈
- 头部玩家很多
- 大家各有优势

## 九、落到输出里的最小要求

如果信息有限，至少输出：
- 二维定位图的两个核心维度
- 主要玩家在图中的位置
- 头部 / 中部 / 长尾判断
- 一句市场格局结论

如果信息较充分，再补：
- 市场份额分布
- 市场类型判断
- 替代方案地图
- 市场空位
- 进入门槛
- 竞争趋势

## 十、使用提醒

- 二维图里的对象不一定都必须是“产品”，也可以是企业、业务线、平台形态或替代方案
- 不要为了好看乱选维度，维度必须能支撑判断
- 如果数据不完整，就明确写成“分层判断”或“趋势判断”
- 市场竞争格局的价值在于帮助用户理解“市场怎么打”，而不只是“有哪些公司”
