# 输出模板

根据不同使用场景，选择不同输出模板。

## 模板一：结论先行版

适合：
- 给老板汇报
- 快速同步结论
- 时间有限的场景

建议结构：
1. 分析目标
2. 核心结论
3. 主要竞品分组
4. 关键差异
5. 机会点
6. 建议动作

写作要求：
- 先讲结论，再讲依据
- 每部分尽量简短
- 强调“对我方意味着什么”

## 模板二：详细拆解版

适合：
- 做系统研究
- 团队内部共识沉淀
- 需要较完整材料的场景

建议结构：
1. 分析背景与目标
2. 目标市场与用户画像
3. 竞品范围与分组
4. 市场关键竞争要素
5. 市场竞争格局
6. 单个竞品拆解
7. 横向对比
8. 关键洞察
9. 建议与下一步

写作要求：
- 保持结构统一
- 每个竞品用同一套卡片
- 对重点竞品补“核心壁垒、盈利模式、收入结构”
- 洞察部分不能只是重复前文
- 重要判断尽量附来源和时间

## 模板三：功能对比版

适合：
- 比较某个功能怎么做
- 支撑功能规划讨论
- 评估是否跟进某项能力

建议结构：
1. 分析目标
2. 对比功能范围
3. 该功能所在环节的关键竞争要素
4. 关键竞品选择
5. 功能与流程对比
6. 各家优劣判断
7. 对我方建议

写作要求：
- 重点写关键流程
- 不要只做功能打勾表
- 说明设计背后的产品意图

## 模板四：市场机会版

适合：
- 新产品立项
- 新市场探索
- 找切入口

建议结构：
1. 分析目标
2. 市场关键竞争要素
3. 市场竞争格局
4. 主要玩家与分层
5. 用户需求满足情况
6. 竞争空白与机会点
7. 切入建议

写作要求：
- 聚焦“市场是否还有机会”
- 强调空白点和切入路径
- 优先用二维定位图和分层判断说明市场结构
- 减少过细功能描述

## 模板五：增长策略版

适合：
- 研究竞品如何获客
- 分析转化路径
- 参考增长打法

建议结构：
1. 分析目标
2. 增长竞争的关键要素
3. 竞品增长路径概览
4. 获客方式对比
5. 转化机制对比
6. 品牌与传播打法
7. 可借鉴点与风险
8. 下一步测试建议

写作要求：
- 关注路径而不是只列渠道
- 解释为什么这种打法适合该产品
- 不要把增长动作脱离产品本身

## 模板六：HTML 报告版

适合：
- 本地预览
- 对外展示
- 团队沉淀
- 希望不是纯文本而是可视化报告

建议结构：
1. 顶部摘要区
2. 分析目标与结论卡片
3. 目标市场区
4. 用户画像区
5. 市场关键竞争要素区
6. 市场竞争格局区
7. 竞品卡片区
8. 横向对比矩阵区
9. 行业发展路径区
10. 机会点与建议区
11. 数据来源区

展示建议：
- 结论用摘要卡片
- 用户画像单独成卡片
- 市场竞争格局区优先展示“坐标轴 + 点”的四象限散点图，以及市场份额饼图和头中尾分层
- 市场份额图下方补一句市场类型判断，例如垄断型、寡头型或分散型
- 对比用表格或矩阵
- 竞品卡片里要体现解决痛点、是否刚需、核心壁垒、盈利模式、收入结构、获客、留存、转化和企业/产品关键路径
- 企业/产品关键路径优先按阶段展示，每阶段至少写主题、决策逻辑、影响和下一步推演
- 行业路径用时间轴
- 不要把整份 HTML 做成纯文本堆叠

## 模板选择原则

- 如果用户强调“汇报”，优先用结论先行版
- 如果用户强调“系统分析”，优先用详细拆解版
- 如果用户强调“某个功能”，优先用功能对比版
- 如果用户强调“立项/机会”，优先用市场机会版
- 如果用户强调“拉新/转化”，优先用增长策略版
- 如果用户强调“展示/本地预览/汇报页面”，优先用 HTML 报告版
