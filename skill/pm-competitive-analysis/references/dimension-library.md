# 分析维度库

不要每次都使用同一套维度。应根据分析目标选择最有价值的维度组合。

## 一、用户维度

适合回答“产品是为谁服务的”。

可用维度：
- 目标用户是谁
- 用户分层是否清晰
- 用户成熟度高低
- 核心痛点是什么
- 使用频率高不高
- 决策者和使用者是否是同一人

## 二、场景维度

适合回答“产品主要解决什么问题”。

可用维度：
- 核心使用场景
- 高频场景还是低频场景
- 单点问题还是完整链路
- 用户使用时机是什么
- 是否具备明显场景优势

## 三、功能维度

适合回答“产品能做什么，做得深不深”。

可用维度：
- 核心功能覆盖
- 功能广度
- 功能深度
- 高阶能力
- 功能组织方式
- 是否存在明显短板

## 四、流程维度

适合回答“用户完成任务是否顺畅”。

可用维度：
- 注册与进入门槛
- 首次价值体验路径
- 核心任务完成步骤
- 关键流程是否清晰
- 阻力点在哪里
- 是否有引导与反馈机制

## 五、体验维度

适合回答“好不好用”。

可用维度：
- 信息架构是否清楚
- 界面复杂度
- 操作学习成本
- 反馈是否及时
- 体验风格偏效率还是偏引导
- 是否适合新手或专业用户

## 六、商业维度

适合回答“怎么赚钱”。

可用维度：
- 定价方式
- 套餐层级
- 免费能力边界
- 付费触发点
- 核心商业模式
- 收费逻辑是否自洽

## 七、增长维度

适合回答“怎么获客和转化”。

可用维度：
- 主要流量来源
- 内容策略
- 活动策略
- 邀请与裂变机制
- 品牌传播方式
- 转化路径设计

## 八、定位维度

适合回答“它想在用户心里占据什么位置”。

可用维度：
- 品牌主张
- 核心卖点
- 面向大众还是专业用户
- 强调低价、效率、品质还是生态
- 与其他竞品的差异化表达

## 九、战略维度

适合回答“它未来可能往哪走”。

可用维度：
- 是否在扩场景
- 是否在补生态
- 是否在提高客单价
- 是否在往平台化发展
- 是否在争夺更高价值用户
- 所属企业的重大决策是否改变产品方向
- 企业是在出海、下沉、高端化还是收缩聚焦
- 产品迭代路径是否反映了组织或战略变化

## 十、市场竞争要素维度

适合回答“这个市场真正靠什么竞争”。

可用维度：
- 用户选择该类产品时最看重什么
- 赛道是更偏价格竞争、效率竞争、体验竞争还是生态竞争
- 各竞品围绕哪些关键因子建立优势
- 不同竞争因子之间是否存在明显权衡
- 哪些竞争因子正在变得越来越重要

常见表达方式：
- 电商：多、快、好、省
- 教育：学习效果、价格、流量渠道、运营效率
- 工具软件：效率、易用性、稳定性、协作性、价格
- 企业服务：能力完整度、实施成本、集成能力、服务能力、品牌信任

## 十一、市场竞争格局维度

适合回答“这个市场里的玩家分别处在什么位置”。

可用维度：
- 二维四象限的两个核心轴是什么
- 不同企业、产品或业务形态分别落在哪个象限
- 头部、中部、长尾如何分层
- 哪些是直接竞品，哪些是替代方案
- 市场份额是否集中
- 哪些位置已经拥挤，哪些位置仍有空位
- 市场的主要进入门槛是什么
- 未来竞争可能往哪个方向变化

常见表达方式：
- 电商：物品丰富度 x 送达时间
- 企业服务：标准化程度 x 行业深度
- 工具产品：单点效率 x 协作闭环
- 医疗产品：服务深度 x 交付复杂度

## 十二、企业/产品关键路径维度

适合回答“为什么这个产品今天会变成这样”。

可用维度：
- 所属企业是谁
- 产品在企业整体业务中的角色
- 当前产品属于企业的增长引擎、战略补位还是生态配套
- 企业/产品的主路径是什么
- 企业的重要战略动作
- 产品的关键版本或功能拐点
- 可以拆成哪些阶段
- 每个阶段的主题是什么
- 每个阶段为什么做这个决定
- 每个阶段暴露了什么业务逻辑
- 每个阶段如何改变产品形态、用户结构或商业模式
- 商业化、出海、下沉、高端化等路径变化
- 哪些外部事件或内部决策推动了产品转向

如果是企业服务类产品，还应补：
- 购买方是谁
- 使用方是谁
- 交付方式是什么
- 如何获客、留存、扩单、续费

## 十三、核心壁垒与商业结构维度

适合回答“它为什么能赢，以及它怎么挣钱”。

可用维度：
- 它解决了什么痛点
- 这个需求是不是刚需
- 行业规模和增速如何
- 真正的核心壁垒是什么
- 壁垒来自数据、网络、资源、政策、技术、品牌还是团队能力
- 为什么只有它更有机会做成
- 商业模式是什么
- 盈利模式是什么
- 是否存在增长闭环或飞轮
- 模式更偏自生型、内耗型还是混合型
- 是否存在明显天花板
- 收入结构由哪些部分组成
- 它当前把资源投在哪些收入方向
- 这些方向上的竞争对手是谁
- 它的资源禀赋是否匹配这些方向

## 常见维度组合建议

### 立项分析

优先组合：
- 市场竞争要素维度
- 市场竞争格局维度
- 核心壁垒与商业结构维度
- 用户维度
- 场景维度
- 定位维度
- 战略维度
- 商业维度

### 功能规划

优先组合：
- 市场竞争要素维度
- 功能维度
- 流程维度
- 体验维度

### 定价评估

优先组合：
- 市场竞争要素维度
- 市场竞争格局维度
- 核心壁垒与商业结构维度
- 商业维度
- 用户维度
- 定位维度

### 增长分析

优先组合：
- 市场竞争要素维度
- 市场竞争格局维度
- 核心壁垒与商业结构维度
- 增长维度
- 流程维度
- 定位维度

### 汇报材料

优先组合：
- 市场竞争要素维度
- 市场竞争格局维度
- 核心壁垒与商业结构维度
- 定位维度
- 功能维度
- 商业维度
- 战略维度
- 企业发展路径维度

## 使用原则

- 优先少而准，不要贪多
- 优先选能支撑结论的维度
- 不要为了完整而牺牲重点
- 如果信息不足，减少维度而不是硬凑内容
- 竞争要素维度和数据佐证维度，通常应优先于细枝末节的功能罗列
